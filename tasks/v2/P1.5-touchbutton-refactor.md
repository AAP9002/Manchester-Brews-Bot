# P1.5 — Refactor TouchButton to use touch.normalize

**Status:** done
**Phase:** P1 — Foundation
**Depends on:** P1.3 (utils/touch.py must exist)
**Blocks:** —
**HLD reference:** §4 (components/TouchButton.py), §7 Touch input model

## Goal
Drop the inline coordinate rotation in `TouchButton` and call `utils/touch.normalize` instead. No behaviour change.

## Context
`components/TouchButton.py` currently has the rotation inlined inside `isPressed`:
```python
tx = raw_touch[1]
ty = 240 - raw_touch[0]
```
After P1.3 lands, `utils/touch.normalize(raw_touch)` returns the same `(tx, ty)` tuple. This task swaps the inline copy for the import.

## Files to touch
- `components/TouchButton.py`

## Specification
1. Add `from utils.touch import normalize` (or equivalent — match the project's import style; `utils.SendRequest` is imported as `from utils.SendRequest import SendRequest` elsewhere). Also import `from utils.config import TIMING`.
2. In `isPressed` replace the two-line inline rotation with `tx, ty = normalize(raw_touch)`.
3. **Add 300 ms tap debounce in `runCallback`/`fire` (whichever the file calls it).** Track `self._next_fire_at = 0.0` in `__init__`; in the fire method:
   ```python
   import time as _t
   now = _t.monotonic()
   if now < self._next_fire_at:
       return
   self._next_fire_at = now + TIMING["tap_debounce"]
   self.callback()
   ```
   This protects against deliberately-rapid sequential presses (especially the back button on AnnouncementScreen). Edge-triggered touch already prevents the same physical press from firing twice; this is the second layer.
4. No other behavioural changes. Keep the padding logic, hidden-flag check, and bounding-box test exactly as they are.

While here, document via a single short comment that the callback parameter must be a callable — do NOT pass the result of a function call. (The bug is fixed at call sites in P1.7, P4.2, P4.3; this is a defensive note.)

```python
# callback must be a callable, not the result of a call. Wrap with `lambda: ...`
# at the call site if you need to pass arguments.
```
