## Goal
A full-screen offline indicator that owns the WiFi reconnect attempts and auto-navigates back to `home` once the device is back online.

## Context
v1's main loop polls WiFi every 5 s and inline-attempts `wifi.radio.connect(...)` if it sees a drop. v2 hoists this responsibility to a dedicated screen so the rest of the codebase doesn't need to know about WiFi state past `app_state["wifi_connected"]`.

The main loop (P4.5) is responsible for **navigating to** `no_connection` when it detects the drop; this screen is responsible for **reconnecting** and navigating back when WiFi is restored.

Reconnect cadence: `TIMING["reconnect_poll"]` seconds (default 5 s).

## Files to touch
- `screens/NoConnectionScreen.py` (create)

## Specification

```python
# screens/NoConnectionScreen.py
import os
import wifi
import displayio
from adafruit_display_shapes.rect import Rect

from utils import layout
from utils.config import COLOURS, MESSAGES, TIMING, DISPLAY


class NoConnectionScreen:
    def __init__(self, navigator, app_state):
        self._navigator = navigator
        self._state = app_state
        self._next_attempt = 0.0

        self._group = displayio.Group()
        self._group.append(Rect(0, 0, DISPLAY["width"], DISPLAY["height"],
                                fill=COLOURS["dark"]))

        # Single big centred message.
        self._group.append(layout.make_text_label(
            MESSAGES["no_connection"], scale=2, color=COLOURS["red"],
            x="center", y="center",
            parent_width=DISPLAY["width"], parent_height=DISPLAY["height"],
        ))

    # ---------- Navigator protocol ----------

    def get_group(self):
        return self._group

    def on_enter(self, params=None):
        self._next_attempt = 0.0  # try immediately

    def handle_touch(self, touch):
        # No interactive elements; tap does nothing.
        pass

    def tick(self, now):
        # If wifi is already up, leave immediately.
        if wifi.radio.connected:
            self._state["wifi_connected"] = True
            self._navigator.navigate("home")
            return

        if now < self._next_attempt:
            return

        self._next_attempt = now + TIMING["reconnect_poll"]

        ssid = os.getenv("CIRCUITPY_WIFI_SSID")
        password = os.getenv("CIRCUITPY_WIFI_PASSWORD")
        if not ssid or not password:
            return

        try:
            wifi.radio.connect(ssid, password)
        except Exception as exc:
            print("reconnect failed:", exc)
            return

        if wifi.radio.connected:
            self._state["wifi_connected"] = True
            self._navigator.navigate("home")
```

The screen owns its own backoff counter (`_next_attempt`) — the main loop's `tick` cadence is high-frequency, so we gate attempts to once per `reconnect_poll`.

## Acceptance criteria
- [ ] `screens/NoConnectionScreen.py` exposes `NoConnectionScreen(navigator, app_state)` implementing the Navigator screen protocol.
- [ ] Renders a single centred "No internet connection" message in red on a dark background.
- [ ] `tick(now)` attempts reconnect at most once per `TIMING["reconnect_poll"]`.
- [ ] On successful reconnect, sets `app_state["wifi_connected"] = True` and navigates to `home`.
- [ ] No exception propagates out of `tick`.
- [ ] No `time.sleep`, no busy-loop.
- [ ] On-device: pull the device's network range, screen appears within 5 s; restore network, screen returns to home within 5 s.

## Out of scope
- A real "back" or "retry now" button — the reconnect happens automatically.
- Spinner / progress animation.
- Captive-portal handling.
- Storing the screen the user was on before the drop and returning there. v2 always returns to `home` post-reconnect.
- Wiring into Navigator at boot or hooking the WiFi-drop branch in the main loop — that's P4.5.
