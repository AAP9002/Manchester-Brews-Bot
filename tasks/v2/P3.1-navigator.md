## Goal
Provide screen routing for v2: a registry of named screens, a method to switch between them with optional parameters, and a `tick(now)` fanout for time-based screen logic (auto-dismiss timers, reconnect polls).

## Context
v1 dispatches screens via if-chains in `code.py` keyed on the string `app_state["current_screen"]`, with `display.root_group = ...` reassignments scattered around. This is brittle and adds friction when introducing a new screen. v2 centralises this in a `Navigator`.

Screens implement a small protocol:
- `get_group()` → `displayio.Group` (root group for `display.root_group`)
- `on_enter(params)` (optional) — called when navigated to, may rebuild
- `handle_touch(touch)` — called per rising-edge touch
- `tick(now)` (optional) — called every loop iteration

## Specification

```python
# utils/Navigator.py
"""Screen registry and routing.

Screens implement (all but get_group are optional):
    get_group()           -> displayio.Group
    on_enter(params=None) -> None
    handle_touch(touch)   -> None
    tick(now)             -> None
"""


class Navigator:
    def __init__(self, display):
        self._display = display
        self._screens = {}
        self._active_name = None
        self._active = None

    def register(self, name, screen):
        if not hasattr(screen, "get_group"):
            raise TypeError("screen must implement get_group()")
        self._screens[name] = screen

    def navigate(self, name, params=None):
        if name not in self._screens:
            raise KeyError("unknown screen: " + repr(name))
        screen = self._screens[name]
        if hasattr(screen, "on_enter"):
            screen.on_enter(params)
        self._display.root_group = screen.get_group()
        self._active_name = name
        self._active = screen

    @property
    def active(self):
        return self._active

    @property
    def active_name(self):
        return self._active_name

    def handle_touch(self, touch):
        if self._active is not None and hasattr(self._active, "handle_touch"):
            self._active.handle_touch(touch)

    def tick(self, now):
        if self._active is not None and hasattr(self._active, "tick"):
            self._active.tick(now)
```

That's the entire surface. No history stack (back button uses `navigate("home")` explicitly), no transition animations.

## Acceptance criteria
- [ ] `utils/Navigator.py` exposes a `Navigator(display)` class with `register`, `navigate`, `active`, `active_name`, `handle_touch`, `tick`.
- [ ] `register` rejects screens without `get_group`.
- [ ] `navigate` raises `KeyError` for unknown names.
- [ ] `navigate` calls `on_enter(params)` only if the screen defines it.
- [ ] `tick` and `handle_touch` are safe to call when no screen is active (no exception).
- [ ] No imports outside the standard CircuitPython runtime.

## Out of scope
- History / back stack — explicit `navigate("home")` is enough for v2.
- Transition animations.
- Any screen wiring — that's P3.2 (with v1 screens) and P4.5 (with v2 screens).
- Concurrency, locks — single-thread.
