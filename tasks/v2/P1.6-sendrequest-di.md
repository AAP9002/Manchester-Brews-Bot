## Goal
1. Drop the class-attribute state injection (`SendRequest.app_state = app_state`) in favour of constructor injection.
2. Change `post`, `get`, `post_sheets` to return `(ok: bool, body: dict | None)` so callers can distinguish "offline" from "request failed".

## Context
`utils/SendRequest.py` currently:
- Owns a module-level `adafruit_requests.Session`.
- Has every method as `@classmethod` reading `cls.app_state["wifi_connected"]`.
- Returns `None` on any failure (offline OR exception).

Read the current file before editing ‚Äî it's only ~68 lines.

The shared HTTP session and `_cleanup` finally pattern stay. The only changes are the constructor and the return shape.

## Specification
1. Convert `SendRequest` from class-method-only into an instantiable class:
   ```python
   class SendRequest:
       def __init__(self, app_state):
           self._state = app_state
           self._session = self._build_session()  # single-session, kept for the lifetime of the instance

       def post(self, url, payload=None, headers=None) -> tuple:
           ...

       def get(self, url) -> tuple:
           ...

       def post_sheets(self, url) -> tuple:
           ...

       def _cleanup(self, response):
           ...

       @staticmethod
       def _build_session():
           ...  # existing socketpool/ssl/session creation
   ```
2. Each public method returns a 2-tuple:
   - `(False, None)` when offline (`self._state["wifi_connected"]` is falsy) ‚Äî no network attempt.
   - `(False, None)` when an exception is caught (preserve the existing print for visibility).
   - `(True, body)` on success, where `body` is the parsed JSON (`None` is acceptable when the response had no JSON body, e.g. some Slack webhook responses).
3. Keep the `gc.collect()` and `response.close()` in `finally`.
4. Remove the class attribute `SendRequest.app_state = app_state` pattern (it will be removed at the call site in P1.7).
