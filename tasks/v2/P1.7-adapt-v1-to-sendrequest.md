## Goal
Update every consumer of `SendRequest` to:
1. Receive a constructor-injected instance instead of using class-attribute state.
2. Handle the new `(ok, body)` tuple return shape.

## Context

The complete list of callers (verify by grepping `SendRequest` in the repo):

- `screens/BroadcastScreen.py` — `SendRequest.post` for brewing/ready webhooks (the helper functions at module top)

## Specification

### `code.py`
Replace the class-attribute injection block with constructor injection:
```python
# was:
SendRequest.app_state = app_state
CoffeeCounter.app_state = app_state
WeatherManager.app_state = app_state

# becomes:
send_request = SendRequest(app_state)
coffee_counter = CoffeeCounter(send_request, app_state)
weather_manager = WeatherManager(send_request, app_state)
```

Pass these instances into screen constructors where needed (or store them on `app_state` if the screens currently look them up that way — match the pattern that minimizes the diff in screens).

### `utils/CoffeeCounter.py`
- Convert from class-method-only into an instantiable class with `__init__(self, send, app_state)`.
- Replace `SendRequest.post_sheets(url)` with `self._send.post_sheets(url)` and unpack `(ok, body)`:
  ```python
  ok, body = self._send.post_sheets(url)
  if not ok or not body:
      return
  count = body.get("count")
  ```
- Same for `get`.
- Keep public method names (`increment`, `sync`, `fetch`, `get_count`).

### `utils/WeatherManager.py`
- Same conversion: instantiable, takes `send` and `app_state`.
- Update `fetch()` to unpack `(ok, body)` from `self._send.get(...)`.
- This file will be deleted in P5.2; do the minimal work to keep it functional in v1.

### `screens/MenuScreen.py`
- The screen calls `SendRequest.post(...)` directly for the low-beans alert. Update the call site to use the injected instance. The simplest path: pass `send_request` to the screen constructor in `code.py` and store it on the screen instance.
- Unpack `(ok, body)` and treat `not ok` the same as v1's `None` return (silent failure with print to serial).

### `screens/BroadcastScreen.py`
- The two module-level helper functions (`send_coffee_brewing_message`, `send_coffee_made_message`) call `SendRequest.post`. Refactor them to accept the `send_request` instance as an argument, or move their logic into the screen class so they can use `self._send`. Either is fine for v1; pick the smaller diff.
- The known callback-at-construction-time bug (`callback=SendRequest.post(WEBHOOK_URL)`) — **do not fix that bug here**. It is fixed in P4.3 when the screen is rewritten. Just adapt the buggy line to the new instance shape; the bug is preserved (still fires at construction) but the imports remain valid.
