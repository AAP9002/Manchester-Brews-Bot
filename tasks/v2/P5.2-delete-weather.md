**Status:** done — pulled forward into `feat/v2-batch-b` on 2026-05-06

**Depends on:** P4.5
**Blocks:** P5.4

> Executed early at user request. Because v1 `MenuScreen` was still in the tree (P5.1 not yet done), this task also stripped the weather widget (`temp_label`, `weather_label`, `rain_label`, `weather_bg`) and the `updateWeather()` method from `screens/MenuScreen.py` — out of P5.2's original scope but unavoidable when running before P5.1.

## Goal
Remove the weather subsystem entirely — no longer used by any v2 screen.

## Context
v2 dropped weather from the home screen design. After P4.5 nothing imports `WeatherManager`, but the file and the related `app_state` keys / `settings.toml` keys still linger. This task purges them.

## Files to touch
- `utils/WeatherManager.py` (delete)
- `code.py` (remove weather-related state, instantiation, refresh cadence)
- `settings.example.toml` (remove `WEATHER_LAT`, `WEATHER_LON`)
- `settings.toml` (optional — leave the user's local `settings.toml` alone unless they ask; the values are inert without `WeatherManager`)

## Specification

1. `git rm utils/WeatherManager.py`.
2. In `code.py`:
   - Remove `from utils.WeatherManager import WeatherManager`.
   - Remove `weather_manager = WeatherManager(...)`.
   - Remove the `weather_temp`, `weather_condition`, `weather_rain_chance` keys from the `app_state` dict initialisation.
   - Remove the 10-minute weather refresh cadence in the main loop.
3. In `settings.example.toml`, delete the `WEATHER_LAT` and `WEATHER_LON` lines.
4. Confirm `grep -r "weather\|WEATHER" .` returns no hits outside docs / tasks / this task itself.

## Acceptance criteria
- [x] `utils/WeatherManager.py` no longer exists.
- [x] `code.py` has no references to `WeatherManager`, `weather_temp`, `weather_condition`, `weather_rain_chance`.
- [x] `settings.example.toml` has no `WEATHER_*` keys.

- Do NOT remove unused BMPs — P5.3.
