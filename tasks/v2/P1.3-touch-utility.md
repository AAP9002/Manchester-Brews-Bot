## Goal
Single source of truth for the CST8xx → display-coordinates rotation.

## Context
The CST8xx capacitive touch driver returns raw coordinates rotated 90° relative to the 320×240 display. Currently the conversion `tx = raw_y; ty = 240 - raw_x` is inlined in `components/TouchButton.py:isPressed`, `screens/MenuScreen.py:isPlusOnePressed` and `isLowBeansPressed`, and a few other places. Read `components/TouchButton.py` for the canonical form.

## Specification
```python
# utils/touch.py
"""Touch coordinate normalization for the 320x240 display.

The CST8xx driver returns coordinates rotated 90 degrees relative to the
display. Every hit-test in this codebase must call normalize() before
comparing against displayio bounds.
"""

DISPLAY_HEIGHT = 240


def normalize(raw_touch):
    """Convert a raw touch tuple to display coordinates.

    raw_touch: (x, y, pressure) or (x, y) as returned by adafruit_cst8xx.
    Returns: (tx, ty) in display coordinates.
    """
    raw_x = raw_touch[0]
    raw_y = raw_touch[1]
    return (raw_y, DISPLAY_HEIGHT - raw_x)
```

Keep the module behaviour-free apart from this function — no imports beyond the standard CircuitPython runtime. No print statements.
