**Status:** done

## Goal
Centralise every magic constant that v2 screens will use: colours, timing, message strings, image paths, display dimensions.

## Context
v1 has hex colours, sleep durations, debounce thresholds and BMP paths sprinkled across `MenuScreen`, `BroadcastScreen`, `WifiIndicator`, `code.py`. v2 needs one module that all new screens import from. Existing v1 screens are NOT being migrated to use this — they will be deleted in P5.1.

Some BMPs referenced (e.g. `announcement.bmp`, `low-beans.bmp`, `log-intake.bmp`) do not exist yet; that's tracked in `open-questions.md`. Reference them by their planned filenames anyway.

## Files to touch
- `utils/config.py` (create)

## Specification
Constants only. No functions, no classes, no behaviour.

```python
# utils/config.py

DISPLAY = {
    "width": 320,
    "height": 240,
}

COLOURS = {
    "blue":    0x2563EB,
    "red":     0xEF4444,
    "teal":    0x14B8A6,
    "success": 0x16A34A,
    "dark":    0x111827,
    "white":   0xFFFFFF,
    "muted":   0x6B7280,
}

TIMING = {
    "success_dismiss": 2.5,    # seconds the SuccessScreen stays before returning home
    "wifi_poll":       5.0,    # seconds between wifi state checks
    "tap_debounce":    0.3,    # min seconds between taps on the same control
    "reconnect_poll":  5.0,    # seconds between reconnect attempts on NoConnectionScreen
}

MESSAGES = {
    # The {name} placeholder is filled at runtime from env var
    # LOW_BEANS_PERSON (default "Someone" if unset). HomeScreen does
    # the .format() — config.py stays constants-only.
    "low_on_beans": (
        "{name} has been peer-pressured effectively...",
        "beans should be with you shortly",
    ),
    "log_intake_default": "Cup logged.",
    "brewing":            "Brewing announced",
    "ready":              "Ready announced",
    "no_connection":      "No internet connection",
}

IMAGES = {
    "send_icon":       "/images/announcement.bmp",
    "low_beans_icon":  "/images/low-beans.bmp",
    "log_intake_icon": "/images/log-intake.bmp",
    "brewing_icon":    "/images/loading.bmp",
    "ready_icon":      "/images/coffee-done.bmp",
    "back_icon":       "/images/back.bmp",
}
```

The `{name}` placeholder is substituted at runtime by `HomeScreen` from `os.getenv("LOW_BEANS_PERSON")`, defaulting to `"Someone"`. `config.py` itself does not import `os` and stays constants-only.

## Acceptance criteria
- [ ] `utils/config.py` exists.
- [ ] All five top-level dicts (`DISPLAY`, `COLOURS`, `TIMING`, `MESSAGES`, `IMAGES`) defined as above.
- [ ] No functions, no classes, no I/O at import time.
- [ ] Module imports cleanly under CircuitPython.

## Out of scope
- Do not migrate any v1 screen to use these constants.
- Do not invent additional constants beyond what is listed — let consumers add what they need when they need it.
- Do not implement environment-variable lookup here. (Webhook URLs continue to come from `os.getenv` at the call site.)
