## Goal
The new announcement screen: two card buttons (Brewing, Ready) plus a small back button bottom-right. Replaces v1's `BroadcastScreen` and **fixes the long-standing callback-at-construction bug**.

## Context
Layout (320×240 display, per the design):
- **Brewing** — left card, blue border, dark fill, brewing icon + label.
- **Ready** — right card, blue border, dark fill, ready icon + label.
- **Back button** — small, bottom-right, uses `back.bmp` via `TouchButton`.

The v1 callback bug:
```python
# v1 — wrong; invokes post() at screen construction time
TouchButton(..., callback=SendRequest.post(WEBHOOK_URL))
```
Must be:
```python
# v2 — correct; pass a callable
TouchButton(..., callback=lambda: send.post(WEBHOOK_URL))
```
Or, since `CardButton.__init__` already raises `TypeError` when `callback` isn't callable, the bug can't recur for the two main cards.

Tap behaviour:
| Element | Action |
|---|---|
| Brewing | `slack.post(send, slack.Msg.BREWING)`; navigate to `success` with `image_path=IMAGES["brewing_icon"]`, `message=MESSAGES["brewing"]`, `return_to="home"` |
| Ready | `slack.post(send, slack.Msg.READY)`; navigate to `success` with `image_path=IMAGES["ready_icon"]`, `message=MESSAGES["ready"]`, `return_to="home"` |
| Back | `navigator.navigate("home")` |

## Specification

```python
# screens/AnnouncementScreen.py
import displayio
from adafruit_display_shapes.rect import Rect

from components.CardButton import CardButton
from components.TouchButton import TouchButton
from utils.config import COLOURS, IMAGES, MESSAGES, DISPLAY
from utils import slack


class AnnouncementScreen:
    def __init__(self, navigator, send):
        self._navigator = navigator
        self._send = send

        self._group = displayio.Group()
        self._group.append(Rect(0, 0, DISPLAY["width"], DISPLAY["height"],
                                fill=COLOURS["dark"]))

        pad = 8
        back_btn_size = 32  # match back.bmp dimensions
        bottom_label_room = 0  # back button sits at bottom-right corner; cards span full available height

        card_h = DISPLAY["height"] - pad * 2
        card_w = (DISPLAY["width"] - pad * 3) // 2

        self._brewing_card = CardButton(
            group=self._group,
            x=pad, y=pad, width=card_w, height=card_h,
            icon_path=IMAGES["brewing_icon"], label="Brewing",
            border_colour=COLOURS["blue"],
            callback=self._fire_brewing,
        )

        self._ready_card = CardButton(
            group=self._group,
            x=pad + card_w + pad, y=pad, width=card_w, height=card_h,
            icon_path=IMAGES["ready_icon"], label="Ready",
            border_colour=COLOURS["blue"],
            callback=self._fire_ready,
        )

        # Back button overlays the bottom-right area of the right card.
        back_x = DISPLAY["width"] - pad - back_btn_size
        back_y = DISPLAY["height"] - pad - back_btn_size
        self._back_btn = TouchButton(
            group=self._group,
            x=back_x, y=back_y,
            image_path=IMAGES["back_icon"],
            callback=lambda: self._navigator.navigate("home"),
            padding=8,
        )

        self._cards = [self._brewing_card, self._ready_card]

    # ---------- Navigator protocol ----------

    def get_group(self):
        return self._group

    def on_enter(self, params=None):
        pass

    def handle_touch(self, touch):
        import time as _t
        now = _t.monotonic()
        # Back button takes priority — it sits visually on top of a card.
        if self._back_btn.is_pressed(touch):
            self._back_btn.fire()
            return
        for card in self._cards:
            if card.is_pressed(touch):
                card.flash(now)
                card.fire()
                return

    def tick(self, now):
        for card in self._cards:
            card.tick(now)

    # ---------- Actions ----------

    def _fire_brewing(self):
        slack.post(self._send, slack.Msg.BREWING)
        self._navigator.navigate("success", {
            "message": MESSAGES["brewing"],
            "image_path": IMAGES["brewing_icon"],
            "return_to": "home",
        })

    def _fire_ready(self):
        slack.post(self._send, slack.Msg.READY)
        self._navigator.navigate("success", {
            "message": MESSAGES["ready"],
            "image_path": IMAGES["ready_icon"],
            "return_to": "home",
        })
```

Brewing and Ready are dispatched via `utils.slack.post(send, slack.Msg.BREWING)` and `slack.post(send, slack.Msg.READY)` — the helper reads the single `SEND_MESSAGE_SLACK_WEBHOOK` env var internally and looks up message content in `utils.config.SLACK_MESSAGES`.

`TouchButton` API names (`image_path`, `is_pressed`, `fire`) may differ slightly from v1 — match the actual file after the P1.5 refactor lands. The intent above is "back button using existing TouchButton with a lambda callback."

## Acceptance criteria
- [ ] `screens/AnnouncementScreen.py` exposes `AnnouncementScreen(navigator, send)` implementing the Navigator screen protocol.
- [ ] Two card buttons (Brewing, Ready) and a small back button.
- [ ] Brewing → posts via `slack.post(send, slack.Msg.BREWING)`, navigates to success with `IMAGES["brewing_icon"]`.
- [ ] Ready → posts via `slack.post(send, slack.Msg.READY)`, navigates to success with `IMAGES["ready_icon"]`.
- [ ] Back → `navigator.navigate("home")`.
- [ ] Callbacks are wrapped lambdas — **no callback-at-construction bug**. CardButton constructor enforces this; the back-button TouchButton uses `lambda: ...`.
- [ ] No `time.sleep` anywhere.
- [ ] On-device: each tap fires exactly once, navigates correctly, success screen returns to home.
